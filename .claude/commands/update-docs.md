---
description: ブランチの変更差分を元に CLAUDE.md・README.md を自動更新する
---

現在のブランチと main の差分を分析して、必要なドキュメントを更新してください。

## 手順

### 1. 差分を確認する
```bash
git diff main...HEAD --name-only
git diff main...HEAD
```

### 2. 変更内容に応じて以下を判断・更新する

**CLAUDE.md を更新すべき変更：**
- 新しいパッケージ追加 → 技術スタック・インストール済みパッケージ欄
- 新しいファイル・ディレクトリ追加 → ディレクトリ規約・コードの状態
- Issue の完了 → 現在の実装状況（完了済み / 次のステップ）
- 環境変数の追加 → 環境変数欄
- app.json の変更 → 注意事項・技術スタック
- ワークフロー・ルールの変更 → 開発ワークフロー欄

**README.md を更新すべき変更：**
- 機能の追加・変更 → 機能説明・セットアップ手順
- 実行コマンドの変更 → Getting Started
- スクリーンショットが必要な UI 変更

### 3. 更新ルール
- **事実のみ記載**：「〜の予定」ではなく「〜済み」「〜実装済み」
- **実装状況の日付**は更新する（`YYYY-MM-DD時点`）
- **削除されたもの**はドキュメントからも削除する
- **軽微な変更**（テスト追加・バグ修正など）は CLAUDE.md の更新不要
- README は**ユーザー向け**（セットアップ・機能説明）、CLAUDE.md は**開発者・AI向け**（実装状況・規約）

### 4. 更新後に確認
変更した箇所を箇条書きで報告する。変更不要と判断した場合もその理由を伝える。
