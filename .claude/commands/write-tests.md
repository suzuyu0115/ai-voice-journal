# /write-tests

## 安全ルール

- シークレット（API キー等）をテストコードにハードコーディングしない
- テスト用の値はモック・フィクスチャで代替する

## 手順

1. 現在の Issue・実装方針を確認する（`/next-action` の出力を参照）

2. テストすべき振る舞いをリストアップする
   - 正常系（期待通りに動くケース）
   - 異常系・エッジケース（エラー・空データ・境界値等）

3. テストファイルを作成する
   - 配置: 対象ソースファイルと同じディレクトリに `*.test.ts` / `*.test.tsx`
   - 形式: Jest + `@testing-library/react-native`（UI コンポーネントの場合）

4. テストを実行して**失敗（RED）**することを確認する
   ```bash
   cd mobile && npm test -- --testPathPattern="対象ファイル名" --no-coverage
   ```

5. ユーザーに以下を提示して承認を待つ：
   - 作成したテストファイルのパス
   - テストケース一覧と各ケースの意図
   - RED になっていることの確認結果（実装前なので失敗が正常）
